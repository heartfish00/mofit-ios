## 판단 요지

"폰 세워두면 자동 카운트 + 로그인 없이 바로 시작 + 무료"라는 컨셉 자체는 내 buy_trigger에 정확히 꽂히지만, **스쿼트 단일 운동 한정 + 정확도 수치 부재 + 앱스토어 리뷰 없음**이라는 세 가지 치명점이 겹쳐 "일단 다운받아볼 동기"가 임계점을 못 넘는다. 설치로 가기엔 증거가 너무 부족해 drop.

## 구체적 이유

- **내 앱 선택 경로가 통째로 빈다**: 나는 앱스토어 별점/리뷰, 인스타 리일 추천을 보고 설치 여부를 결정한다. 문서 §5-4에서 본인들이 "앱스토어 출시 전 단계 · 별점·리뷰 없음 · README 스크린샷도 없음"을 고지함. 즉 **내 의사결정 필수 재료가 0**이다. 세일즈맨이 직접 링크를 주는 방식 자체가 내 습관과 어긋남.
- **rep 카운트 정확도 수치가 "아직 없다"**: 내 가장 강한 reject_trigger는 "한 번이라도 misdetect 하면 즉시 삭제"인데, 문서 §5-1이 "오카운트율·미검출률 수치 공개 미제공 + 내부 실패 케이스 로깅 파이프라인도 아직 없음"을 솔직히 밝힘. 솔직함은 점수지만, 정작 내가 확인할 방법은 **내 몸으로 3일 써보는 것 외엔 없음**. 보수 성향 + 세일즈맨 신뢰 40인 나에겐 부담.
- **"스쿼트 전용"이 너무 좁다**: §1.3 + §5-3에서 푸쉬업·런지·플랭크 **영구 미제공 가능성**까지 고지. 나는 맨몸 운동 레퍼토리 전체(스쿼트/푸쉬업/플랭크/런지)를 돌리고 싶은 홈트 초급자다. 스쿼트 카운터 하나 때문에 별도 앱을 띄우는 일상적 마찰은 3일차 이탈의 정답.
- **AI 코칭 차별점은 로그인 + 7일 뒤에나 확인 가능**: §5-8 — 비로그인 샘플 2종은 하드코딩 정적 카피. 내가 실제로 받는 피드백 퀄리티는 "이메일·비밀번호 가입 + 7일간 꾸준히 운동"이라는 내 3일 컷 테스트보다 한참 먼 뒤에야 검증됨. "그거면 ChatGPT 쓰지" 결론 가능성 상존.
- **비로그인 기록이 로그인 시 증발**: §5-5 (ADR-013). "3일 써보고 마음에 들면 로그인"이라는 자연스러운 내 유입 흐름에서, 로그인하는 순간 기록이 초기화된 것처럼 보이는 UX. 불신 포인트.
- **온보딩 3~4분 · 6단계**: 내 기준 "첫 30초 안에 뭔가 되어야 함"에 비하면 길다. 성별/키/몸무게까지는 수용 가능하나 "체형·목표·코치 스타일"까지 6단계면 첫 카운트까지 심리적 거리가 있음. §1의 "1분 안에 첫 카운트"와 §2.1의 "온보딩 3~4분"이 살짝 어긋남.
- **프로덕트 성숙도 증거 부재 (감점)**: UX probe 리포트나 UI 스크린샷이 이 가치제안에 동봉되지 않음. 문서상 기능 명세는 풍성하지만 실제 브랜드 식별성·빈 상태·공유용 자산·에러 UI의 완성도를 검증할 방법이 없어 **신뢰도 추가 감점**. 내 tech_literacy가 medium이라 "어느 정도는 내가 보면 알지" 수준인데, 그 "보여줄 것"이 지금 없음.

## 걱정/의문점

- "권장 환경: 밝은 실내, 2~3m, 일상복"의 반대 조건(저녁 집 조명, 거실 작은 공간 1.5m 이하, 헐렁한 트레이닝복)에서 실제로 얼마나 버티는가? 내 집 환경이 권장 환경에 얼마나 가까운지 감이 안 온다.

- "수개월 후 과금 후보 영역 · 아직 확정 아님"이 얼마나 아직인가? 내가 6개월 뒤에 "코칭 탭 유료화"로 놀라게 될 확률은?
- 스쿼트 전용 포지셔닝이 ADR로 확정되었다는데, 그럼 내가 이 앱을 설치해야 할 이유가 "스쿼트 카운팅 하나에 앱 하나 쓰겠다"는 결심으로 좁혀지는 것 아닌가? 그 결심을 정당화할 만큼 rep 카운팅이 정확하다는 증거가 필요하다 — 그게 없으니 현재로선 drop이 자연스럽다.
