# 가치제안 개선 포인트

단일 세션이라 빈도가 아니라 키맨의 `구체적 이유` + `걱정/의문점`에서 군집화한 주제.

1. **rep 카운트 정량 정확도 증거 (최상위 차단 요소)** — 대표 발화: *"내 가장 강한 reject_trigger는 '한 번이라도 misdetect 하면 즉시 삭제'인데 … 수치 공개 미제공 + 내부 실패 케이스 로깅 파이프라인도 아직 없음. 솔직함은 점수지만, 정작 내가 확인할 방법은 내 몸으로 3일 써보는 것 외엔 없음."* 제안: 베타 대기 말고 **내부 자체 측정(5~10명 × 권장/비권장 환경 각 20세트)이라도 숫자를 붙여라.** "정확도 미공개"는 트러스트 40 페르소나에겐 "검증 불능"과 동의어.
2. **외부 신뢰 신호 전면 부재 (앱스토어 리뷰·별점·스크린샷·UX probe)** — 대표 발화: *"세일즈맨이 직접 링크를 주는 방식 자체가 내 습관과 어긋남 … 내 의사결정 필수 재료가 0이다."* 제안: TestFlight 베타 테스터 후기 3~5건 + 데모 GIF + 스크린샷 1페이지라도 랜딩에 동봉. 소비자 페르소나는 "문서 완성도"가 아니라 "남들이 쓰는 증거"를 본다.
3. **스쿼트 전용 포지셔닝의 설치 정당성** — 대표 발화: *"스쿼트 카운터 하나 때문에 별도 앱을 띄우는 일상적 마찰은 3일차 이탈의 정답."* 제안: "스쿼트 전용 = 정확도 세계1위"라는 **포지셔닝의 대가**를 구체화하거나, 혹은 **"스쿼트 먼저, 푸쉬업 N주 내"**의 짧은 타임라인을 약속해 홈트 레퍼토리 미스핏을 부분 해소.
4. **AI 코칭 차별점 검증 지연 (로그인 + 7일)** — 대표 발화: *"'그거면 ChatGPT 쓰지' 결론 가능성 상존."* 제안: 비로그인 샘플을 정적 하드코딩이 아니라 **온보딩 입력값(키/몸무게/목표) 기반으로 동적 생성**해서, 첫 10분 안에 "generic하지 않음"을 체감시켜라. 현재 §2.5의 2종 하드코딩 카드는 ChatGPT 대비 증거가 되지 못함.
5. **비로그인 → 로그인 기록 미이관 UX (ADR-013)** — 대표 발화: *"3일 써보고 마음에 들면 로그인이라는 자연스러운 내 유입 흐름에서, 로그인하는 순간 기록이 초기화된 것처럼 보이는 UX. 불신 포인트."* 제안: 이관을 하거나, **"기록 탭은 로컬 소유권 유지 + 서버는 추가 분석 레이어"** 라는 설명을 로그인 전에 먼저 납득시켜라.
6. **온보딩 길이 vs 첫 카운트 시간 모순** — 대표 발화: *"§1 '1분 안에 첫 카운트' vs §2.1 '온보딩 3~4분' — 실제 첫 rep까지 시간은 몇 분인가?"* 제안: 온보딩 skip 옵션 or 카피 정합성 수정. "1분"은 첫 설치 유저의 기대를 만드는 숫자이므로 깨지면 그 자체가 신뢰 손실.
7. **환경 조건 민감도 투명 공개** — 대표 발화: *"권장 환경의 반대 조건(저녁 집 조명, 거실 1.5m 이하, 헐렁한 트레이닝복)에서 실제로 얼마나 버티는가?"* 제안: 환경 매트릭스(거리 × 조도 × 복장) 자체 테스트 결과를 FAQ에 공개. 정확도 수치 부재를 부분적으로라도 보완.
